# DDL、多数据源与扩展边界

## 自动维护 DDL

- DDL 会改变数据库结构，属于高风险操作；未明确批准不要执行。
- 使用前确认目标环境、迁移工具、备份、回滚脚本、发布窗口和权限。
- 自动 DDL 不应替代正式迁移流程；生产环境优先走 Flyway、Liquibase 或项目既有变更流程。
- 生成或执行 DDL 前必须确认实体字段、索引、默认值、注释、字符集和历史数据兼容。

## 多数据源

- MyBatis-Plus 自身不是完整动态数据源治理方案；优先复用项目已有多数据源组件。
- 每个 `SqlSessionFactory` 是否都注册了 `MybatisPlusInterceptor`、TypeHandler、Mapper XML 路径，需要分别确认。
- 跨数据源事务、读写分离、路由键来源属于高风险范围，不能只改 Mapper 注解。

## 高级扩展

- 自定义全局配置、SQL 注入器、插件、TypeHandler、ID 生成器都会影响多个模块。
- 扩展前先确认是否有局部解决方案；确需全局扩展时写清影响范围和回滚点。
- 插件顺序会影响 SQL 改写结果，尤其是租户、动态表名、数据权限、分页和非法 SQL 拦截。
